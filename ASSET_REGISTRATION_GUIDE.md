# `menu_assist` 플러그인 에셋 등록 및 관리 가이드 (v3)

이 문서는 `menu_assist` 플러그인의 아이콘 데이터를 관리하는 **3가지 방법**을 안내합니다. 개발자가 아니어도 UI와 구글 시트를 통해 손쉽게 에셋을 등록할 수 있습니다.

---

## 1. 개요: 어떤 방식을 써야 하나요?

| 방식 | 추천 상황 | 데이터 저장 위치 | 특징 |
| :--- | :--- | :--- | :--- |
| **① UI로 바로 등록** | 개인 작업, 소량의 아이콘 추가 | 현재 Figma 파일 내부 (`pluginData`) | 가장 빠름. 다른 파일에는 공유되지 않음. |
| **② Google Sheets 연동** | **[권장]** 팀 전체 공유, 대량 관리 | 구글 스프레드시트 (외부) & Figma 코드 | 엑셀처럼 관리 가능. 팀원 모두에게 자동 동기화. |
| **③ 코드 직접 수정** | 개발자가 초기 데이터 구축 시 | `data/icon-db.ts` 파일 | 가장 안정적이나, 코드 수정 후 배포 필요. |

---

## 2. [방식 ①] UI로 바로 등록하기 (플러그인 내부)

플러그인 실행 후 **'에셋 관리'** 탭에서 진행합니다.

1.  **데이터 입력**:
    *   키워드, 설명, 컨셉(유의어)을 입력합니다.
2.  **Component Set 가져오기**:
    *   Figma 캔버스에서 아이콘 Component Set을 선택합니다.
    *   **"Figma에서 선택한 Component Set 가져오기"** 버튼을 누릅니다.
    *   자동으로 이름과 Key가 입력되고, **Variant 속성 목록**이 체크박스로 뜹니다.
3.  **Variant 선택**:
    *   등록하고 싶은 아이콘 옵션만 체크합니다. (코드를 몰라도 됩니다!)
4.  **등록**:
    *   "✅ 에셋 등록" 버튼을 누르면 즉시 플러그인에 반영됩니다.

> **참고**: 이렇게 등록한 데이터는 **현재 Figma 파일**에만 저장됩니다. 백업하려면 **"📥 JSON 내보내기"**를 이용하세요.

---

## 3. [방식 ②] Google Sheets 연동 (팀 공유용)

코드를 건드리지 않고, 구글 시트를 통해 팀원들과 아이콘 목록을 공유하는 방법입니다.

### 3-1. 구글 시트 준비하기
1.  새 구글 스프레드시트를 만듭니다.
2.  첫 번째 줄(헤더)에 다음 **6개항목**을 정확히 입력합니다:
    *   `keyword` (필수)
    *   `description`
    *   `concept`
    *   `componentSetName` (필수)
    *   `componentSetKey` (필수)
    *   `variants` (JSON 형식, 예: `[{"type":"badge"}]`)
3.  데이터를 채워 넣습니다.

### 3-2. 웹에 게시 (CSV)
1.  구글 시트 메뉴에서 **[파일] > [공유] > [웹에 게시]**를 클릭합니다.
2.  **"전체 문서"** 대신 해당 시트를 선택하고, **"웹 페이지"** 대신 **"쉼표로 구분된 값(.csv)"**을 선택합니다.
3.  생성된 링크를 복사합니다.

### 3-3. 플러그인에서 동기화
1.  플러그인 **'에셋 관리'** 탭 하단의 **"구글 시트 연동"** 섹션으로 이동합니다.
2.  입력창에 복사한 **CSV 링크**를 붙여넣습니다.
3.  **"🔄 구글 시트와 동기화"** 버튼을 누릅니다.
4.  "✅ 00개의 에셋을 동기화했습니다!" 메시지가 뜨면 완료입니다.

---

## 4. [방식 ③] 코드 직접 수정 (개발자용)

기존 v2 방식입니다. `data/icon-db.ts` 파일을 직접 수정하여 영구적인 기본 데이터를 구축할 때 사용합니다.

1.  `ICON_COMPONENT_SET_KEY` 객체에 컴포넌트 셋 Key 추가.
2.  `iconData` 객체에 키워드와 속성 정보 추가.
3.  `npm run build` 실행.

---

## 5. 데이터 백업 및 내보내기

*   **JSON 내보내기**: UI에서 등록한 커스텀 데이터를 `.json` 파일로 다운로드할 수 있습니다. 이를 개발자에게 전달하여 `icon-db.ts`에 영구 반영 요청을 할 수 있습니다. Note: This file is a specific file that the user requested updates for.
